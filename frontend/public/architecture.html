Wayfinder Supply Co. Architecture

System Overview

Wayfinder Supply Co. demonstrates Elastic Agentic Search by federating data from multiple sources and using AI agents to create personalized recommendations.

Component Details

Frontend (React + Vite)

Technology: React 18, TypeScript, Tailwind CSS, Radix UI

Features: Server-Sent Events for real-time agent responses, thought trace visualization, user persona switching

Backend Proxy (FastAPI)

Responsibilities: Proxy SSE streams from Agent Builder, product catalog queries, cart management

Endpoints: POST /api/chat, GET /api/products, GET /api/products/search, POST /api/cart

Agent Builder

Trip Planner Agent: Main orchestrator that uses multiple tools (search, workflows, ES|QL) to make intelligent decisions

Trip Itinerary Agent: Synthesizes trip plans and formats markdown output with loyalty perks

Workflows

check_trip_safety: Calls MCP server for weather conditions and road alerts

get_customer_profile: Calls MCP server for CRM data (purchase history, loyalty tier)

get_user_affinity: ES|QL query on clickstream to analyze user preferences

Elasticsearch Indices

product-catalog: Products with semantic_text field (ELSER), hybrid search (keyword + semantic), rich metadata

user-clickstream: User browsing behavior, tag preferences, used for personalization

MCP Server (FastAPI)

Purpose: Simulates external enterprise systems (CRM, Weather API)

Protocol: JSON-RPC over HTTP

Tools: get_trip_conditions_tool, get_customer_profile_tool

Data Flow

Chat Request Flow

  1. User sends message via frontend
  2. Frontend streams to backend /api/chat
  3. Backend proxies to Agent Builder /api/agent_builder/converse/async
  4. Agent Builder processes:
    • Calls workflow tools (weather, CRM)
    • Executes ES|QL queries (affinity)
    • Searches product catalog
    • Calls Itinerary Agent
  5. Responses stream back via SSE
  6. Frontend displays in chat + thought trace
